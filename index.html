INTRODUCTION

저는 강사로서의 경험을 통해 접한 불편함과 이에 대한 해결책을 찾는 과정에서 개발자로의 전환을 결심하게 되었습니다. 강사로 일하면서 학생들을 가르치고 도와주는 과정에서 학생들을 위한 모의고사나 학습 자료를 만들어야 했습니다. 그러나 기존의 프로그램이나 툴을 사용할 때 문제가 많았습니다. 문제 은행을 활용한 모의고사 생성 프로그램에서는 중복 처리가 제대로 이루어지지 않거나, 범위 설정이 제대로 되지 않는 등의 오류가 발생했습니다. 이러한 불편함을 느끼면서 “어떻게 하면 이런 오류를 없앨 수 있을까?”, “이런 기능이 추가로 있다면 좋겠다.” 는 생각을 자주 하게 되었습니다. 이러한 경험을 통해 직접 문제를 해결하고, 더 나은 솔루션을 제공하기 위해 프로그래밍을 배우기를 결심했습니다. 개발자로서의 역량을 키워 이러한 문제들을 해결할 수 있고, 더 나은 서비스를 제공할 수 있는 사람이 되고자 합니다.

PROJECT

앱 오픈마켓 웹 사이트(Open Appstore)

바로가기

2024. 02

• 프로젝트 개요
: Open Appstore는 개인 개발자와 사용자를 연결하는 플랫폼 웹 사이트로, 개발자는 자신의 앱을 업로드하여 판매할 수 있고, 사용자는 업로드된 앱을 수요에 맞게 구매하고 별점을 남길 수 있습니다.

• 프로젝트 참여 인원 및 기간
: 4명, 1주(2024. 02. 19 ~ 2024. 02. 23)

• 프로젝트 개발 환경
◦ 주요 개발 과정 : Eclipse의 Dynamic Web Project(JSP), MySQL 데이터베이스, Apache Tomcat 이용

◦ 프론트 디자인 : Tailwind CSS 프레임워크 이용

• 수행 업무
◦ 사용자 및 개발자 관리 기능 : 회원의 정보를 입력받아 데이터베이스에 저장하고, 저장된 데이터를 기반으로 로그인 세션을 생성하여 사용 가능한 범위 내에서 사이트를 이용할 수 있도록 구현하였습니다.

◦ 앱 관리 기능 : 개발자는 앱을 등록, 수정, 삭제할 수 있도록 하고, 사용자는 등록된 앱을 카테고리별, 최신순 등의 형태로 조회할 수 있도록 구현하였습니다.

◦ 구매 기능 : 사용자가 등록된 앱을 선택하면 가상의 결제가 발생하고 각 기록이 개발자와 사용자 측에 남아 조회되도록 구현하였습니다.

◦ 별점 기능 : 사용자가 본인의 구매 기록을 조회하는 항목에서 GUI를 통해 해당 앱에 대한 별점을 부여할 수 있도록 구현하였습니다.

• 경험
: 프로젝트를 통해 백엔드 개발자로서 기본적으로 요구되는 프론트엔드와 데이터베이스 사이의 요청을 처리하는 작업을 경험했습니다. 프로젝트를 진행하면서 외부에서 정보를 입력받을 때, 의도치 않은 형식이나 내용의 데이터가 입력되면 처리가 제대로 이루어지지 않았던 적이 있었는데, 통해 데이터를 입력 받을 때 데이터 무결성 검사를 진행하는 백엔드 로직에 대해 팀원들과 함께 고민하여 해결할 수 있었습니다.

화장품 제조 공정 시각화 프로그램(Cosmocare)

바로가기

2024. 04

• 프로젝트 개요
: Cosmocare는 화장품 제조 공정간 측정되는 물리 데이터를 저장, 관리하고 시각화하여 보여주는 품질관리 보조 프로그램입니다.

• 프로젝트 참여 인원 및 기간
: 4명, 2주(2024. 04. 01 ~ 2024. 04. 12)

• 프로젝트 개발 환경
◦ 주요 개발 과정 : Visual Studio의 .NET 프레임워크(C#), MSSQL 데이터베이스 이용

• 수행 업무
◦ 데이터 관리 및 조회 기능 : 사용자가 물리 데이터를 등록, 수정, 삭제할 수 있도록 하고 메인 화면에서 표 형식으로 조회할 수 있도록 구현하였습니다.

◦ 사용자 경험 개선 : 데이터베이스에서 데이터가 호출이 완료될 때까지 프론트에서 아무 반응이 없어 프로그램이 정상 작동 상태인지, 오류가 발생한 상태인지를 파악할 수 없었고, 이를 개선하기 위해 데이터 호출과 UI 출력을 분리하고 로딩바를 도입하여 명확히 프로그램 상태를 인지할 수 있도록 구현하였습니다.

• 경험
: 약 6만 개의 데이터를 시각화하는 과정에서 아무리 코드를 살펴보아도 이유를 알 수 없는 잦은 오류가 발생했던 경험이 있었습니다. 그리고 고민 끝에 데이터베이스에서 데이터를 호출하는 백엔드 부분이 성능적으로 좋지 않아 호출된 데이터를 기반으로 시각화를 하는 프론트엔드 부분이 오류를 일으킨다는 것을 알게 되었습니다. 따라서 백엔드 부분과 프론트엔드 부분을 스레드로 나누어 구현하여 해결하였고, 이를 통해 백엔드 로직에서 기능적인 부분만 고려할 것이 아닌, 비기능적인 부분도 프로그램의 안정성에 많은 영향을 준다는 것을 배울 수 있었습니다.

• 프로젝트 참여 인원 및 기간

◦ 전체 UI 설계 및 구현 : 각 팀원이 맡아 제작한 시각화자료를 메인 페이지에 연결하고 회의에서 결정한 전체 UI 설계도에 따라 배치하는 역할을 수행하였습니다.

• 경험
: 팀장으로 프로젝트를 진행하면서, 처음에는 전체 UI의 설계 및 구현 부분을 담당하였습니다. 프로젝트는 역할 배분 후 단기 목표를 달성해나가고 이를 주 단위로 상호 점검하는 애자일 방식으로 진행했습니다. 그러던 중 차트 시각화를 담당한 팀원이 버튼을 이용한 동적 업데이트 기능을 구현함에 있어 어려움을 겪고 있고 해당 기능을 목표 기한까지 달성할 수 없다고 판단하여, 협의 하에 제가 그 작업을 대신 담당하여 프로젝트를 완료한 경험을 하였습니다. 이를 통해 저는 전체 팀의 목표를 달성하기 위해 개인의 역할을 넘어서 적극적으로 문제를 해결하는 책임감과 팀장으로서 팀원의 문제를 인지하고 필요한 조치를 취하는 리더십을 기를 수 있었습니다.
